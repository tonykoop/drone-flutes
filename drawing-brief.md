# Drawing Brief — Drone Flute Family

Quality bar: Fender Stratocaster blueprint level. Drawings live in `drawings/`. Generated by `scripts/generate_drawings.py` v4 (or hand-built for the initial set — see `drawings/build_drawings.py` in this repo).

## Current authority status

The checked-in `drawings/*.svg` files are review previews, not released shop
authority. No body, bore, finger-hole, SAC/window, drone-block, socket, fixture,
or CAM-ready DXF/CAD export is present in the repo as of the Round 33C/D shop
handoff slice.

Decorative DXFs under `inlay-patterns/` are separated from acoustic and core
fabrication authority. They may be used for decorative pocket/plug planning
after a facet datum is confirmed, but they do not control bore length, hole
placement, window geometry, drone-block tuning, or body fit.

See `dxf-shop-authority.md` and `visual-output-register.csv` before treating
any visual or DXF artifact as shop evidence.

## Per-member drawings

One SVG per family member, plus one shared assembly drawing.

### `drawings/{member}.svg` (4 files)

Single-page tabloid (11×17) layout. Sections:

1. **Title block (bottom right)** — instrument name, member ID, key, date (2026-05-05), revision (R0), units (inches), scale (1:2), source workbook, source CAD, drawn-by.
2. **Plan view** — flute lying on its side, bird block on top facet, full length visible. Show window position, all 6 finger holes with diameters, foot end, drone socket.
3. **Side view** — rotated 45°, shows the octagonal cross-section in projection.
4. **Section A-A** — through-bore axial section. Bore ID, wall thickness, SAC chamber, splitting edge wall.
5. **Section B-B** — radial section through the body, showing octagonal facets and bore.
6. **Hole schedule callout** — small table inset listing each hole's distance from foot, diameter, undercut angle.
7. **Datum chain** — primary datum at FOOT (open end), secondary at WINDOW (splitting edge), tertiary at SOCKET (drone block face).
8. **Tolerances** — tuning-critical (hole positions ±0.020"), fit-critical (drone tenon ±0.003"), shop tolerances (everything else ±0.030").

### `drawings/assembly.svg` (1 file)

Exploded view of one flute (use F#m as the canonical) with callouts for body, bird block, leather lace, drone block set (3 blocks fanned).

### `drawings/family-overview.svg` (1 file)

All 4 flutes in plan view side-by-side, scaled to a common foot datum. Shows the proportional relationship between keys.

## Generation method

For now, hand-built via `drawings/build_drawings.py` in this repo. When v4's `generate_drawings.py` becomes the canonical tool, switch to it.

## Cross-reference table

Each critical dimension on each drawing must trace back to one of:
- A formula in `Drone-Flutes-Design.xlsx` (cite as `=Master_Inputs!B7` or similar).
- A row in `family-spec.csv`.
- An assumption explicitly noted on the drawing (e.g., "facet inset 0.10" — assumed").
